Este documento detalha o DNA técnico e visual do projeto "Simulador Arquitetônico", servindo como um guia mestre para reconstrução e desenvolvimento.

1. DNA VISUAL (Design System)

O design system é construído sobre Tailwind CSS com uma estética minimalista e funcional, utilizando o sistema de temas do Radix UI.

2. DNA FUNCIONAL (User Experience)

Fluxo do Usuário

  1. Abertura: O usuário abre o aplicativo e a interface principal do Simulador Arquitetônico é carregada.
  2. Visualização: Uma área de canvas central exibe uma planta arquitetônica.
  3. Interação: O usuário interage com os controles da UI para modificar parâmetros da planta.
  4. Processamento: O plantaEngine.ts recalcula a estrutura da planta.
  5. Renderização: O plantaRenderer.ts desenha a planta atualizada no canvas.
  6. Feedback: O usuário vê a planta modificada em tempo real.

Mapeamento de Rotas

O roteamento é gerenciado pela biblioteca wouter.

  • / (Raiz): Renderiza o componente principal App.
  • Qualquer outra rota: Renderiza a página "404 Not Found".

4. DNA DE NEGÓCIO (Core Purpose)

Objetivo do Aplicativo

Fornecer uma ferramenta de simulação arquitetônica que permite aos usuários gerar, visualizar e manipular plantas baixas de forma interativa.

Usuário Final

Profissionais ou estudantes da área de arquitetura, design de interiores ou engenharia civil que necessitam de uma ferramenta para prototipação visual rápida.
